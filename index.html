Sakura Tree

Using Python's Turtle object and recursion, I've written a program that draws a sakura tree. Originally the tree was perfectly symetrical, which made it look unnatural. To solve this I added some randomness (branch length, branching probability, branch angle, branch width, etc...). The result was something a little closer to nature. My first draft mismanaged memory by creating a lot of unneeded instances of the Turtle Object. Thanks to John Leszczynski's elegant code (which passes only one instance of the Turtle object through the call stack) I was able to refactor and greatly simplify my solution.
